## Getting Your Account

### Initial Setup

The IT team will provide you with:
- **Email:** Your ministry email (e.g., alicmd.youth@gmail.com)
- **Temporary Password:** A secure password
- **Login URL:** The calendar platform URL

**Important:** Change your password on first login!

### Changing Your Password

1. Log in with your temporary credentials
2. Click on your email in the top right
3. Go to Account Settings
4. Click "Change Password"
5. Enter a strong new password
6. Click "Update Password"

## For Contributors

As a ministry leader (contributor), you can create and manage events for your ministry.

### Creating an Event

1. **Click "Create Event"** button (top right)

2. **Fill in the event details:**
   - **Title:** Name of your event (e.g., "Youth Group Meeting")
   - **Description:** Details about the event
   - **Room:** Select which room you need (Room A, B, or C)
   - **Start Date & Time:** When the event begins
   - **End Date & Time:** When the event ends

3. **Save as Draft or Submit:**
   - **Save as Draft:** Keep working on it later
   - **Submit for Review:** Send to admin for approval

### Event Status Explained

Your events can have different statuses:

| Status | What it means | What you can do |
|--------|---------------|-----------------|

### Editing Your Events

1. **Click on your event** in the calendar
2. **Make your changes**
3. **Save** (only works for drafts)

**Note:** You can only edit events that are in "Draft" status. Once submitted, contact an admin to make changes.

## For Admins

As an admin, you have additional responsibilities and capabilities.

### Reviewing Events

1. **Go to "Admin Panel"** (in navigation menu)

2. **See Pending Events:**
   - Review each event submission
   - Check for conflicts
   - Verify details are complete

3. **Take Action:**
   - ‚úÖ **Approve:** Event is ready to publish
   - ‚ùå **Reject:** Event needs changes (add notes)
   - üëÅÔ∏è **View Details:** See full event information

### Publishing Events

1. **Go to "Admin Panel"**
2. **Click "Approved" tab**
3. **Click "Publish Event"** on approved events
4. Event now appears on public calendar

### Managing Rooms

1. **Go to "Rooms"** (in navigation menu)

2. **Add New Room:**
   - Click "Add Room"
   - Enter name, description
   - Choose a color
   - Set as Active
   - Click "Create Room"

3. **Edit Room:**
   - Click edit icon on room card
   - Update details
   - Click "Update Room"

4. **Deactivate Room:**
   - Click edit icon
   - Toggle "Active" to off
   - Room won't appear in event creation

### Managing Users

1. **Go to "Users"** (in navigation menu)
2. **View all users and their roles**
3. **Import new users from CSV** (if IT provides)

### Admin Checklist

Daily:
- [ ] Review pending events
- [ ] Approve/reject submissions
- [ ] Publish approved events

Weekly:
- [ ] Check for scheduling conflicts
- [ ] Review upcoming events
- [ ] Ensure rooms are properly configured

Monthly:
- [ ] Review user accounts
- [ ] Check for inactive users

## FAQ

### General Questions

**Q: I forgot my password. What do I do?**
A: Contact the IT team at alicmd.@gmail.com to reset your password.

**Q: Can I change my email address?**
A: Contact the IT team. Your email is tied to your ministry role.

**Q: How do I see the public calendar?**
A: Visit the `/public` page (no login required). This is what the community sees.

### For Contributors

**Q: Why can't I edit my event after submitting?**
A: Once submitted for review, only admins can make changes. This ensures quality control. If you need changes, contact an admin.

**Q: How long does approval take?**
A: Typically 1-3 days. For urgent events, contact an admin directly.

**Q: Can I delete an event?**
A: Only admins can delete events. Contact an admin if you need to cancel.

**Q: Why was my event rejected?**
A: Check the reviewer notes. Common reasons:
- Incomplete information
- Room conflict
- Timing issues
- Need more details

**Q: Can I book multiple rooms?**
A: You need to create separate events for each room.

### For Admins

**Q: Can I edit events after publishing?**
A: Yes, but changes will be visible immediately to the public. Edit carefully.

**Q: What if two events overlap?**
A: The system prevents this automatically. You'll get an error if trying to approve overlapping events in the same room.

**Q: Can I unpublish an event?**
A: Yes, edit the event and change status back to "approved" or "draft".

**Q: How do I handle recurring events?**
A: Currently, create separate events for each occurrence. Recurring events feature may be added later.

### Technical Issues

**Q: The page won't load**
A:
1. Check your internet connection
2. Try refreshing the page (Ctrl+R or Cmd+R)
3. Clear browser cache
4. Try a different browser
5. Contact IT if problem persists

**Q: I see an error message**
A:
1. Note the exact error text
2. Try the action again
3. If it persists, screenshot the error
4. Contact IT with the screenshot

**Q: Can I use this on my phone?**
A: Yes! The platform is fully responsive and works on phones and tablets.

## Tips for Success

### For Everyone

1. **Be Descriptive:** Use clear titles and detailed descriptions
2. **Plan Ahead:** Submit events early
3. **Check Calendar:** View existing events before scheduling
4. **Update Status:** Keep event information current
5. **Communicate:** Contact admins or IT when you need help

### For Contributors

1. **Think About Your Audience:** Write descriptions that inform and invite
2. **Double-Check Details:** Verify date, time, and room before submitting
3. **Follow Up:** Check status of your submitted events
4. **Be Patient:** Allow time for admin review

### For Admins

1. **Review Promptly:** Check pending events daily
2. **Provide Feedback:** When rejecting, explain why
3. **Stay Organized:** Keep track of approved vs. published events
4. **Communicate:** Let contributors know about their submissions
5. **Monitor Conflicts:** Watch for scheduling issues
